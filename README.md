### Aiti Guru - тестовое задание (Frontend React)

Приложение по тестовому заданию для Aiti Guru: экран авторизации и панель с товарами на основе DummyJSON Products/Auth. Есть вход с “запомнить меня”, таблица с сортировкой и пагинацией, поиск по API, добавление товара через модалку и визуальное соответствие макету из Figma.

---

### Стек

- **React 18+**: `react`, `react-dom`
- **TypeScript**: строгий режим (`strict: true`)
- **Сборка**: Vite
- **Роутинг**: `react-router-dom`
- **Состояние**: `zustand`
- **Запросы / кеширование**: `@tanstack/react-query`
- **Формы и валидация**: `react-hook-form` + кастомные хелперы
- **UI**: Tailwind CSS 4, компоненты в стиле `shadcn/ui`, `sonner` для toast-уведомлений
- **API**: `https://dummyjson.com` (Auth + Products), проксируется через `/api`

---

### Функциональность

- **Экран авторизации**
  - Валидация логина и пароля (обязательность, минимальная/максимальная длина).
  - Обработка ошибок API: при неуспешном ответе берётся `message` из DummyJSON (если есть) и показывается пользователю.
  - Чекбокс **“Запомнить данные”**:
    - при включённом чекбоксе токены и флаг хранятся в `localStorage` (сессия живёт после закрытия браузера),
    - при выключенном - данные хранятся в `sessionStorage` (сброс при закрытии вкладки).
  - Защита роутов: без авторизации переход на `/` редиректит на `/login`, при наличии сессии - сразу открывается дэшборд.

- **Страница с товарами**
  - Загрузка данных из DummyJSON Products (`/products` и `/products/search`) через React Query и обёртку `authFetch`.
  - Таблица соответствует колонкам из макета: наименование, вендор, артикул, рейтинг, цена, действия.
  - При загрузке/рефече поверх страницы показывается прогресс-бар, таблица визуально “подгружается” (blur/opacity).
  - **Сортировка** по столбцам (рейтинг, цена и т.д.) - состояние сортировки и страница хранятся в query-параметрах (`sortBy`, `order`, `page`).
  - **Поиск** по товарам через API (`/products/search`) с дебаунсом ввода и синхронизацией состояния в URL (`q`).
  - **Добавление товара**:
    - модалка с формой: наименование (`title`), цена (`price`), вендор (`brand`), артикул (`sku`), рейтинг (`rating` опционально),
    - базовая валидация (обязательные поля, формат цены/рейтинга),
    - при успешном добавлении показывается toast и новый товар попадает в начало текущего списка.
  - **Подсветка рейтинга**: если рейтинг товара \< 3, значение подсвечивается красным.

### Запуск через Docker

Собрать образ:

```bash
docker build -t aiti-guru-frontend .
```

Запустить контейнер:

```bash
docker run -p 8080:80 aiti-guru-frontend
```

Приложение будет доступно на `http://localhost:8080`.

---

### Локальный запуск (без Docker)

Требования: Node.js 20+, pnpm (или npm/yarn).

```bash
pnpm install
pnpm dev
```

По умолчанию приложение будет доступно на `http://localhost:5173`. Проксирование запросов к DummyJSON настроено через `vite.config.ts` (`/api` → `https://dummyjson.com`).

Скрипты:

- **`pnpm dev`** - режим разработки
- **`pnpm build`** - продакшен-сборка
- **`pnpm preview`** - предпросмотр собранного билда
- **`pnpm lint`** - запуск ESLint
- **`pnpm typecheck`** - проверка типов TypeScript


### Тестовые данные DummyJSON

Для проверки авторизации можно использовать тестовые учётные данные из DummyJSON (например):

- **Логин**: `evelyns`
- **Пароль**: `evelynspass`

Актуальный список тестовых пользователей: в разделе Auth на сайте DummyJSON.
